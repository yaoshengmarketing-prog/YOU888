# 娛樂城系列網站 — 母版說明（Master Template Guide）

> 這份文件是「開新站的設計圖與說明書」。任何一站（YOU888、78OK、以及未來 3A、88WIN、UBET88、RG富遊…）都從這套母版複製，只換皮、不改架構。
> 新的 Cowork 對話只要從 GitHub 抓下 `you888-site` 母版，連同這份說明一起讀，就能零損失接手，不需要看過先前的聊天記錄。

---

## 一、技術架構

- **框架**：Astro（靜態產生器 SSG）
- **部署**：GitHub（原始碼）→ Netlify 自動 build（`npm run build`）→ 上線
- **自動發文**：新增一個 `.md` 檔到 `src/content/blog/` → 推上 GitHub → Netlify 自動重建，文章即上線
- **關鍵檔案**
  - `src/consts.ts`：品牌名稱、網域、站台標題、8 大分類與 Hub 設定（**改這裡＝換站**）
  - `public/images/`：banner 封面圖
  - `src/content/blog/*.md`：文章（frontmatter 含 title / description / category / tags / pubDate / faq…）
  - `src/pages/`：頁面與路由；`src/components/SiteHeader.astro`：統一抬頭；`src/styles/global.css`：全站樣式
- **網址結構（本 Astro 站慣例，固定不可亂改）**
  - 文章：`/post/{slug}/`
  - 分類：`/blog/categories/{category-slug}/`
  - 部落格總覽：`/blog/`
  - Hub 頁：`/deposit-guide/`、`/withdraw-guide/`
  - 自動產生：`/sitemap.xml`、`/rss.xml`、`/robots.txt`

## 三、8 大分類（固定，複製到每一站）

| slug | 顯示名稱 |
|---|---|
| platform-guide | 平台指南 |
| register-account | 註冊與帳號 |
| app-mobile | APP與手機版 |
| promotions | 優惠活動 |

| game-guide | 遊戲玩法 |
| security | 安全查核 |
| customer-service | 客服與常見問題 |

原則：
- 一個分類 ＝ 一個獨立、清楚的搜尋／瀏覽需求（對準玩家真實搜尋意圖）。
- 分類內用 **Hub ＋ Topic Cluster** 深化，**不濫用 tag**（避免薄頁與索引膨脹）。
- 「存款與出金」分類下有兩個 SEO Hub 頁：**存款教學總整理**、**出金／提款問題總整理**。

## 四、SEO 慣例（每站沿用，只換品牌字與數字）

- **Meta Title 格式**：`{品牌} 娛樂城｜體育投注・真人・電子捕魚棋牌・每日優惠`
- **Meta Description**：一句話帶「多元遊戲 ＋ 主要優惠 ＋ 存取款 ＋ 客服」，含品牌名，約 60–80 字。
- **結構化資料**：文章用 `BlogPosting` JSON-LD；分類／Hub 用 `WebPage`；FAQ 區用 `FAQPage`；每頁含 `canonical`。
- **Sitemap／RSS／robots**：自動產生，網域一律以 `consts.ts` 的 `SITE.url` 為準。
- **內連原則**：只連「確定存在」的頁；不確定是否存在的目標不放進正文，改列「待確認內連清單」。
- **首頁層級**：8 格分類 → 最新文章 → 小型「查看全部文章 →」連結。
  - 部落格總覽頁保留，但**不做成第九個大型入口**；**抬頭也不放「部落格」連結**。
- **抬頭**：LOGO（唯一回首頁點）＋ 8 分類 ＋ 登入／註冊（每頁都有）＋ 手機漢堡選單。

## 五、開一個新站的步驟（Recipe）

1. 到「該站的本機專案資料夾」開一個新的 Cowork 任務，並把該資料夾連進去。
2. 貼上開場白，請 Claude 從 GitHub `yaoshengmarketing-prog/you888` 的 `you888-site` 抓母版當基底（不要從零寫），並一起讀這份 `母版說明.md`。
3. **換皮**：
   - `consts.ts`：`SITE.name / brand / url / blogTitle / blogDesc` 改成新站
   - 主色：`global.css` 與首頁 inline style 的金色 → 新站主色
   - Logo：從新站現有線上站或其 GitHub 抓（通常不必人工上傳）
   - Banner：放進 `public/images`（可沿用新站現有線上 banner）
   - 優惠文案：改成新站自己的金額／活動
4. **內容**：文章大同小異可沿用改寫，金額／活動換成新站的。
5. **部署**：推到「該站自己的」GitHub repo → Netlify 自動 build 上線。
6. **檔案分站存放**：每站檔案各自存在自己的本機資料夾，互不混用。

## 六、注意事項

- 登入／註冊按鈕 URL 依各站實際填入；未提供前先用 `#` 佔位。
- **高風險動作**（發布上線、刪除資料、付款、寄送訊息、改帳號或安全設定）在按最後一步前先用一句話確認。
- 一律使用繁體中文（台灣用語）溝通。
- 指定使用 Browser 1 操作瀏覽器；找不到指定瀏覽器或多台無法判斷時，停下來詢問，不亂連。
